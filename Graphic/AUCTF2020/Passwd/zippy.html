AUCTF 2020 - Password Cracking - Zippy

----------------------------------------------------------------
Zippy
Have fun!
NOTE: The flag is NOT in the standard auctf{} format
----------------------------------------------------------------

We download the zip associated with the challenge.
We try to unzip it, but a password is requested.
First, I use a python script with the rockyou wordlist to try to break the password. Unfortunately, it doesn't work.
So I decide to use zip2john, which is a tool associated with john the ripper, which will allow to find the password hash.
It works, we recover the password hash.
So I use after hashcat to break this hash:

hashcat -a 0 -m 0 zippy.hash big.txt

Hashcat finds the password after a few seconds: 8297018229.
So I go unzip zippy.zip, I enter the password, and there surprise: A second zip that also asks for a password!
So I reuse the previous technique, and as you can expect, I get another zip with another password.
I repeat this operation until I get to flag.txt.
We get our flag: y0ud1d17#15789
